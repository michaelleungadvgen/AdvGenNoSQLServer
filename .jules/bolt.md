## 2026-02-07 - Avoid recalculating O(N) invariant variables in nested loops
**Learning:** During BM25 scoring inside `FullTextIndex`, `GetAverageDocumentLength()` was dynamically invoked for *each* matching document for *each* query token. Because `GetAverageDocumentLength()` iterated over all indexed documents (O(N) operation), this effectively transformed a fast inverted index search into a catastrophic O(T × P × D) bottleneck. Furthermore, `_documents.Values` was being utilized to acquire the average, eagerly locking and allocating the entire dataset array under concurrent conditions.
**Action:** When working on search or read-heavy paths, analyze which values represent invariants for that specific read context and hoist them out of the loops. Always prefer direct explicit iteration (`foreach (var kvp in dictionary)`) for computing aggregates against concurrent structures instead of their `.Values` LINQ operations. For high-volume aggregates like Average Document Length, consider tracking the running sums on insert/remove instead of runtime evaluation.
