## 2026-10-27 - Optimize ConcurrentDictionary Iteration in ShardingManager
**Learning:** Calling `.Values` or `.Values.ToList()` on a `ConcurrentDictionary` notoriously acquires all locks across all buckets and eagerly allocates memory, causing O(N) memory allocations and unnecessary lock pressure. Multiple LINQ queries (like `.Sum()`, `.Average()`, `.Min()`, `.Max()`) on these values perform redundant linear scans. Furthermore, using `.Count` is less efficient than checking `.IsEmpty` when just testing for the presence of elements.
**Action:** Always iterate directly over the `ConcurrentDictionary` using `foreach (var kvp in dictionary)` and calculate all aggregates within a single pass to eliminate O(N) allocations and redundant scans. Replace `.Count == 0` or `.Count > 0` with `.IsEmpty` property checks.
